What's new in HAT 5.1
Note:
  • If the tool fails to obtain authorization to capture network traffic, please run the script 'Uninstall_TKCaptureAssistant.sh' bundled in the HAT zip file. If any additional errors occur, please contact your MFi representative.
General:
  • Added support for Apple TV Remotes accessory category.
  • Added support for HomeKit Data Stream.
  • Resolved issue that intermittently prevented HAT from seeing the first frame from an IP camera stream.
  • Resolved issue that caused the accessory discovery progress bar to never complete.
  • HAT will now display the Complete Local Name for BLE accessories.
Known Issues:
  • Controllers assigned to shared keystores are not visible after relaunching HAT; please recreate the controllers and assign them back to the shared keystore when necessary.
  • iCloud Relay: Test with one iCloud Relay controller; using multiple controllers is not supported.
  • Switching between BLE controllers while connected is not supported. Manually disconnect from the active controller using the disconnect button before using a different controller.
  • TCB011 is not supported; HAT does not support Pair-Resume to the same accessory from multiple BLE controllers.
  • BLE characteristic values are not updated when an Encrypted Broadcast Notification is received.
  • If an IP accessory is not visible in HAT after performing WAC, restart the controller's Bonjour discovery.
  • In some cases, HAT may hang when attempting to quit the application after using iCloud Relay; please save before force-quitting the application.
  • Event color highlighting may be incorrect when an EVENT is sent to the controller after an HTTP request and before the response.
Requirements:
  • HAT 5.1 requires macOS Mojave 10.14 or later (latest release recommended), which is available for download from the App Store.
Bug Reporting:
  • Please report all issues to bugreport.apple.com by selecting the HomeKit Accessory Development product, and choosing the Certification Tools problem area.
